Track only what matters.

## Character Basics

Write:

- Candidate Name
- Archetype
- Loop Number

## Core Stats

### Battery

- Range: 0 to 100
- Represents available energy
- Usually decreases by 5 per resolved objective
- If Battery reaches 0, hard reboot triggers

### Stability

- Range: 0 to 100
- Represents how stable your simulation state is
- Changes after most checks based on dice count
- If Stability reaches 0, game over

### SIP

- Starts at 0
- Gain +1 after each completed loop
- Spend 1 to negate next OS die modifier
- Persists across loops

## Dice Pools

Assign one of each value: 1, 2, 3

- MSK (Masking): social performance and expected behavior
- SNS (Sensory): body load, noise, regulation
- LOG (Logic): systems, routines, analysis

Archetype presets:

- Social Camouflage: MSK 3, SNS 1, LOG 2
- System Strategist: MSK 1, SNS 2, LOG 3
- Tension Runner: MSK 2, SNS 3, LOG 1

## Objective Tracking

For each objective, note:

- Objective symbol and name
- Pool used
- Player dice rolled
- OS modifier
- Total result
- Outcome: Success or Glitch

## Loop Totals

Track cumulative:

- Total Successes
- Total Glitches
- Final Battery at loop end
- Final Stability at loop end

## Suggested Sheet Sections

Minimum layout:

- Identity block
- Pools block
- Resources block (Battery, Stability, SIP)
- Objective log table
- Loop receipt notes

## What to Update Every Objective

Fast checklist:

- Resolve roll or special check
- Update Stability if needed
- Apply Battery cost
- Check for Battery 0 hard reset
- Check for Stability 0 game over
- Advance to next objective
